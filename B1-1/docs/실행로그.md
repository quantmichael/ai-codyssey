# 실행 로그

# 1. 시나리오

AI 고객 문의 챗봇 구축 프로젝트의 회의록을 기반으로 프로젝트 관리 문서를 작성하는 업무를 수행하였다. 회의록에서 회의 목적, 결정사항, Action Items, 리스크, 확인이 필요한 사항을 추출하고, 프롬프트를 단계적으로 개선(System Prompt, Few-shot, CoT)하여 결과 품질을 향상시켰다. 또한 Hallucination 검증과 10턴 이상의 대화를 통해 문맥 유지와 조건 변경 반영 능력을 평가하였다.

# 3. 문제 지점과 개선 요약

초기 결과에서는 회의록에서 제안하거나 논의된 내용을 일부 **결정사항으로 해석**하는 문제가 있었다. 또한 리스크가 하나의 목록으로 작성되어 일정, 보안, 비용 측면을 구분하기 어려웠다. 이후 사용자 요청에 따라 리스크를 유형별로 재구성하고, 추가 정보를 반영하여 서버 사용 방식을 수정하였다. 마지막으로 회의록에 없는 내용이나 확정되지 않은 사항을 다시 검토하여 **결정사항과 확인 필요 항목을 명확히 구분**함으로써 문서의 정확성과 실무 활용성을 향상시켰다.
